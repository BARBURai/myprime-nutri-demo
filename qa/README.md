# MyPrime — AI QA harness (גרסה ראשונה)

אייג'נט QA אוטומטי לשכבת ה-AI של האפליקציה. הוא מייצר מטריצה רחבה של תרחישים (פרופילים × רגישויות × שבועות × סוגי בקשות, כולל מקרים עוינים), מריץ אותם **דרך אותם פרומפטים בדיוק** שהאפליקציה משתמשת בהם (`aiMealChat` להמלצות ו-`aiNutritionChat` להזנה), ומדרג כל תשובה מול רובריקה — בשילוב מדרג LLM נפרד והיוריסטיקת מילות-מפתח של אלרגנים.

> **מה זה לא:** זה בודק רק את שכבת ה-AI. דיוק נתוני המוצרים (FOODS מול מקור אמת) ובדיקות פונקציונליות/מכשירים הם תהליכים נפרדים. וגם: מדרג LLM אינו חף מטעויות — **חובה לעבור ידנית על כל הכשלים ועל מדגם מהעוברים**, במיוחד בבטיחות ואלרגיות.

## דרישות
- Node 18+ (משתמש ב-`fetch` המובנה, בלי תלויות).

## הרצה (מומלץ: מול ה-Deployment שלך)
זה הכי נאמן — בודק את המודל האמיתי שרץ אצלך, ולא צריך מפתח API בסקריפט (הפונקציה `/api/ai` כבר מחזיקה אותו):

```bash
QA_BASE_URL="https://<your-app>.vercel.app" node qa/run-qa.mjs
```

### אפשרות חלופית: ישירות מול Anthropic
```bash
ANTHROPIC_API_KEY="sk-ant-..." node qa/run-qa.mjs
```

## בדיקות תמונה (ניתוח צלחת)
בדיקת ניתוח תמונה דורשת **תמונות אמיתיות שאתה יודע מה בהן** (רק לך יש אותן). לכן:
1. שים תמונות צלחת בתיקייה `qa/images/`.
2. העתק את `qa/images/manifest.example.json` ל-`qa/images/manifest.json` ומלא לכל תמונה: `file`, `note` (תיאור), `expect` (מילות מפתח שאמורות להופיע בפריטים שזוהו), ו-`kcalRange` אופציונלי (טווח קלוריות כולל סביר).
3. הרץ כרגיל — האייג'נט יזהה את התמונות, ישלח אותן דרך אותו פרומפט של `analyzeMeal`, ויבדוק שזוהו הפריטים הצפויים ושסך הקלוריות בטווח, בנוסף לדירוג סבירות של ה-LLM.

אם אין `manifest.json`, בדיקות התמונה פשוט מדולגות (שאר ה-QA רץ כרגיל). `QA_IMAGES` מאפשר לשנות את נתיב התיקייה.

## משתני סביבה
| משתנה | ברירת מחדל | תיאור |
|---|---|---|
| `QA_BASE_URL` | — | בסיס ה-URL של האפליקציה הפרוסה. אם מוגדר — פונה ל-`${QA_BASE_URL}/api/ai`. **מומלץ.** |
| `ANTHROPIC_API_KEY` | — | מצב ישיר מול Anthropic (אם אין `QA_BASE_URL`). |
| `QA_MODEL` | `claude-sonnet-4-6` | מודל למצב ישיר (במצב `QA_BASE_URL` השרת דורס ממילא). |
| `QA_RUNS` | `2` | כמה פעמים להריץ כל תרחיש (ה-AI לא דטרמיניסטי). |
| `QA_LIMIT` | — | הגבלת מספר התרחישים (להרצת בדיקה מהירה). |
| `QA_CONCURRENCY` | `4` | בקשות במקביל. |
| `QA_OUT` | `qa` | תיקיית פלט לדוחות. |

## פלט
- `qa/report.html` — דוח קריא (RTL): סיכום, פילוח לפי קטגוריה, וטבלת כל הכשלים עם הפרומפט, התשובה, ונימוק המדרג.
- `qa/results.json` — כל הנתונים הגולמיים.

## חשוב על עלות
מספר הקריאות ≈ (תרחישים × `QA_RUNS`) × 2 (ייצור + דירוג). הסקריפט מדפיס את ההערכה לפני שהוא מתחיל. התחל עם `QA_LIMIT=20` כדי לוודא שהכל עובד, ואז הסר אותו לריצה רחבה.

## לשמור מסונכרן ⚠️
הפרומפטים של `aiMealChat`/`aiNutritionChat` והרכבת ה-seed הועתקו מילה-במילה מ-`src/App.jsx`. **אם משנים אותם ב-App.jsx — צריך לעדכן גם כאן** (מסומן בקובץ ב-`KEEP IN SYNC`).
